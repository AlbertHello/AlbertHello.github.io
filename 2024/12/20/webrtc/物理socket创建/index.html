一、Socket 的创建时机

BasicPortAllocatorSession 执行 OnAllocate task 并进入 DoAllocate 时才会开始创建 socket。

创建 socket 的调用堆栈

  • BasicPortAllocatorSession::OnAllocate

    • BasicPortAllocatorSession::DoAllocate

      • AllocationSequence // 构造 sequence

      • AllocationSequence::Init

        • BasicPacketSocketFactory::CreateUdpSocket // 开始创建 udp socket

          • SocketFactory::Create

            • SocketDispatcher() // 构造SocketDispatcher

            • SocketDispatcher::Create

              • PhysicalSocket::Create // 调用系统 API 创建真正的 socket

              • SocketDispatcher::Initialize()

                • 这里主要做两件事：

                • 1、设置socket异步模式

                • 2、创建的socket放在socketserver中管理

          • BasicPacketSocketFactory::BindSocket // 入参之一是一个本地地址

            • PhysicalSocket::Bind // 将上面刚刚创建的 socket 绑定到本地 IP 和端口

详细描述

  • 创建的 socket 有好几层封装：

    • 具体的物理 socket 是由 PhysicalSocket 创建

    • SocketDispatcher 继承 PhysicalSocket，也就持有了 PhysicalSocket 创建的 socket 的句柄

    • 再创建一个AsyncUDPSocket，因为AsyncUDPSocket正好持有SocketDispatcher

    • 所以 AllocationSequence::init

      • 1、函数中创建的 udp_socket_ 对象就是 AsyncUDPSocket 对象；

      • 2、同时设置好数据的回调函数：AsyncUDPSocket 收包后通过 RegisterReceivedPacketCallback 回调到 sequence 的 OnReadPacket 函数。

二、Socket 的核心相关类

  • Socket 工厂类有两个PhysicalSocketServer和NullSocketServer，顾名思义两者的区别就是PhysicalSocketServer 有具体的物理 socket。

  • 创建的具体的物理 socket 类是：PhysicalSocket，创建、销毁、读、写都是在该类中实现。

  • SocketDispatcher 继承 PhysicalSocket

  • AsyncUDPSocket 又是对SocketDispatcher的封装

所以这几个核心类的关系：

  • AsyncUDPSocket —持有—> SocketDispatcher

  • SocketDispatcher —继承—> PhysicalSocket —创建—> socket
