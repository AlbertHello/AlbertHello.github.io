webrtc之PeerConnection创建

本文最后更新于 2026-06-23T17:29:53+08:00

在创建 peer_connection 之前需要先创建 peer_connection_factory： webrtc 之创建 peer_connection_factory

1、创建PeerConnection的接口

接口文件是 api/peer_connection_interface.h。

M146 推荐使用 CreatePeerConnectionOrError：

其中 RTCConfiguration 和 PeerConnectionObserver 是最重要的两个参数。

  • RTCConfiguration： PeerConnection的全局配置项。全局配置项是提供给WebRTC内部使用的参数信息，可以通过参数来控制WebRTC的内部逻辑、行为方式

  • PeerConnectionObserver：是 PeerConnection 的事件回调，应用层通过实现这些回调方法来作出自己想要实现的逻辑。

2、创建 PeerConnection

触发函数：pc_factory->CreatePeerConnectionOrError(config, dependencies);

可见是调用的 pc_factory 的函数。在 webrtc 之创建 peer_connection_factory中已经解释了，pc_factory 其实是一个代理类对象，最终会转到 PeerConnectionFactory::CreatePeerConnectionOrError。

sequenceDiagram
    participant App as 应用层
    participant Proxy as PeerConnectionFactoryProxy
    participant Factory as PeerConnectionFactory
    participant Alloc as BasicPortAllocator
    participant PC as PeerConnection

    App->>Proxy: CreatePeerConnectionOrError(config, dependencies)
    Proxy->>Factory: 切到 signaling thread
    Factory->>Factory: 校验 RTCConfiguration
    Factory->>Factory: ParseAndValidateIceServers
    Factory->>Alloc: new BasicPortAllocator
    Factory->>Factory: worker_thread BlockingCall(CreateCall_w)
    Factory->>PC: PeerConnection::Create(...)
    PC->>PC: InitializeNetworkThread
    PC->>PC: 创建 SdpOfferAnswerHandler
    Factory-->>App: PeerConnectionProxy

其最终实现在peer_connection_factory.cpp中：

  • 创建 BasicPortAllocator，用于遍历网卡，收集 ip 和 端口，必须在网络线程创建

    • 接收两个主要参数：default_network_manager和default_socket_factory，这都是在创建pc_factory_时创建的。

  • 做一些 BasicPortAllocator的初始化工作，函数是SetNetworkIgnoreMask，参数是network_ignore_mask，默认忽略回还地址。

    • ADAPTER_TYPE_ETHERNET：以太网

    • ADAPTER_TYPE_WIFI：无线WIFI网络

    • ADAPTER_TYPE_CELLULAR：蜂窝网络(2g,3g,4g,5g)

    • ADAPTER_TYPE_VPN：VPN

    • ADAPTER_TYPE_LOOPBACK：回环地址

  • 构造 PeerConnection 对象

  • PeerConnection 构造函数里会调用 InitializeNetworkThread

  • 最终返回 PeerConnection 的代理类对象 PeerConnectionProxy

2.1 、PeerConnection

核心创建函数如下：

PeerConnection 是 WebRTC 会话对象的核心。M146 中大量底层状态通过 callback + PostTask 从 network thread 回到 signaling thread。

上面的英文注释翻译如下：

  • 一方面PeerConnection单独提供如下功能：

    • 管理WebRTC的会话状态（信令状态）；

    • 创建和初始化WebRTC内部的低层次对象/底层对象，比如PortAllocator、BaseChannels；

    • 持有RtpSender/RtpReceiver、track对象，并管理它们的生命周期；

    • 跟踪当前的local/remote sdp。

  • 一方面PeerConnection与其他对象一起提供如下功能：

    • 解析并翻译sdp

    • 根据当前状态生成offer sdp和answer sdp；

    • 管理ICE状态机

    • 产生统计日志

2.2、PeerConnection 构造函数

该函数逻辑比较多，主要逻辑有如下几个：

  • 创建 sdp 会话 session id，也就是 SDP 的 O 行，比如：o=- 3875060266275821690 2 IN IP4 127.0.0.1

  • stun/turn server 相关处理

  • 创建 JsepTransportController，并通过 callback + PostTask 把网络线程上的 ICE、candidate、连接状态回调到 PeerConnection。

  • 创建 sdp_handler_，管理所有 sdp 的解析和创建

  • 创建 RtpTransmissionManager，管理 transceiver、sender、receiver。

2.3、InitializeNetworkThread

  • JsepTransportController 持有 port_allocator_，后面 SetLocalDescription 时会创建 ICE/DTLS/RTP transport。

  • ICE candidate、ICE 状态、candidate pair 变化等网络线程事件，都会通过 lambda 再 PostTask 回 signaling thread。

2.4、InitializePortAllocator_n

  • 这里是 BasicPortAllocator 真正初始化的位置。

  • 默认开启 shared socket、IPv6、Wi-Fi IPv6；如果配置禁用 TCP、高成本网络、link-local network，就在这里追加对应 flag。

  • SetConfiguration 把 STUN/TURN server、候选池大小、TURN 裁剪策略等信息保存到 allocator。后面 candidate 收集时，BasicPortAllocatorSession 会读取这些配置。
