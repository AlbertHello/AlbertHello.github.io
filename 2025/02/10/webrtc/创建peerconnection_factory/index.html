webrtc之PeerConnectionFactory创建

本文最后更新于 2026-06-23T17:29:40+08:00

关于 rtc 第一步就是创建 PeerConnectionFactory，然后再由这个工厂对象去创建实际的 PeerConnection。今天只讲 PeerConnectionFactory 的创建。

1、创建 PeerConnectionFactory 的接口

接口文件是create_peerconnection_factory.h

M146 主线这里对外就是这一个核心接口，可选参数集中到默认参数和 PeerConnectionFactoryDependencies 中。

  • network_thread：网络线程。webrtc 三大线程之一，处理网络数据报的收发相关工作。一般传入 null，由内部在peer_connection_factory.cc 中创建，webrtc 线程是另外一个知识点，请移步 webrtc 之线程模型。

  • worker_thread：工作线程。webrtc 三大线程之一。一般传入 null，由内部在peer_connection_factory.cc 中创建，大部分的工作都是在该线程运行。

  • signaling_thread：信令线程。webrtc 三大线程之一。主要用来与应用层交互。移动端常见做法是把当前业务线程包装成 webrtc 的 Thread，让应用层 API 和 signaling thread 对齐。

  • field_trials：用于创建 Environment，后面 PeerConnection、网络、媒体模块都会通过 env 读取 field trial。

  • rtc::scoped_refptr：这是webrtc 的引用计数系统，又是另外的知识点，移步到 webrtc 之引用计数系统

2、创建 PeerConnectionFactory 的调用堆栈

  • 应用层调用 webrtc::CreatePeerConnectionFactory()

    • CreatePeerConnectionFactory() api/create_peerconnection_factory.cc

      • CreateModularPeerConnectionFactory() api/create_modular_peer_connection_factory.cc

        • PeerConnectionFactory::Create() pc/peer_connection_factory.cc

          • ConnectionContext::Create()

          • new PeerConnectionFactory

        • PeerConnectionFactoryProxy::Create(...)

2.1、CreatePeerConnectionFactory()

  • 该函数主要是把外部传入的线程、音频设备、编解码器工厂等对象整理进 PeerConnectionFactoryDependencies。

  • 如果调用方传入了 network_thread，M146 会把 network_thread->socketserver() 放进 dependencies，后面默认 socket factory 会用到。

  • EnableMedia(dependencies) 会把音视频相关依赖组合成 media factory。

  • 真正创建工厂对象的动作在 CreateModularPeerConnectionFactory() 里。

  • PeerConnectionFactory 必须创建在 signaling thread 上。如果当前线程不对，M146 用 BlockingCall 切到 signaling thread。

  • 最终返回的不是裸的 PeerConnectionFactory，而是 PeerConnectionFactoryProxy。

2.2、ConnectionContext::Create()

  • ConnectionContext 是给 peerconnection 提供资源的，什么资源？三大线程、网络管理类、socket 管理类。

  • Create 函数会创建 ConnectionContext 对象，并且在其构造函数中对三大线程做一些处理。

  • 通过 CreateWithSocketServer() 创建网络线程 network_thread_，说明该线程带有 socket server，可以处理网络 IO 操作，并且 Start，开启线程消息循环。

  • 通过 Thread::Create() 创建 worker_thread_ 线程。该线程不是 socket server 线程，主要处理媒体、Call、编码相关工作。然后 Start，开启线程的消息循环，等待处理 task。

  • 创建了网络管理对象 BasicNetworkManager，创建 socket 工厂类对象 BasicPacketSocketFactory，至于这两个对象有什么作用，这又是另一个知识点，参考 webrtc 之网络建立

2.3、PeerConnectionFactory::Create()

  • M146 中这里没有单独的 Initialize() 主流程。ConnectionContext 创建好后，直接构造 PeerConnectionFactory。

  • PeerConnectionFactory 持有 ConnectionContext，因此也间接持有三大线程、BasicNetworkManager、BasicPacketSocketFactory、media engine 等对象。

2.4、return PeerConnectionFactoryProxy::Create()

  • 这是创建 PeerConnectionFactory 最终返回的结果。

  • 返回的是 PeerConnectionFactory 的代理类 PeerConnectionFactoryProxy。

  • 应用层拿到的是代理对象，代理对象内部持有真正的 pc_factory。

  • 应用层调用有关 PeerConnectionFactory 的方法时，先进入代理类，再由代理类切到正确线程，最后调用真正的 PeerConnectionFactory。

  • 代理类，也叫接口宏，之所以这么设计是为了解决线程乱入并且达到代码隔离的效果，为的是不让应用层的使用不规范污染到 WebRTC 核心代码。有关接口宏请参考 webrtc 之接口宏
