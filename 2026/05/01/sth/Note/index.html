git命令拉取代码出错

拉取代码时报错：
error: The following untracked working tree files would be overwritten by merge
此时仓库内在Tower上看没有任何改动，但git pull后有如下显示：

解决办法：
git clean -d fx “相应目录”
相关命令：
git clean -f -n：查看会删除哪些文件
git clean -f：删除上一条命令显示出来的文件
git clean -fd：删除文件夹
git clean -fX：删除已被忽略的文件
git clean -fx：删除已被忽略和未被忽略的文件

mac charles 抓包https包

1 在Proxy - SSL Proxying Settings打开ssl设置，添加需要抓包的地址，点击add后在host栏添加*号，代表抓所有地址，在port栏添加*号,代表抓所有端口，也可以设置具体的host和port。

2 在Help - SSL Proxying - Install Root Charles Certificate，在mac钥匙串了找到Charles证书，设置完全信任
3 在Help - SSL Proxying - Install Root Charles Certificate On Mobile Device Or Remote Browser，按照弹出提示操作：先在手机上设置好代理，然后去手机端Safari上输入chls.pro/ssl，安装证书并且信任。
4 这是抓包时看到https地址下显示Unkonwn，让然显示不了内容，这和iOS系统版本有关系。iOS10以后的版本不仅需要把Charles证书信任，还得去设置里 通用 - 关于本机 - 证书信任设置 - 打开对Charles的信任。 两处信任才能抓Https的包。

SSH 相关

  • open ~/.ssh/ 查看当前ssh key
  • 创建ssh密钥。使用 ssh-keygen 命令生成密钥对（推荐 RSA 算法，4096 位强度）：
    • ssh-keygen -t rsa -b 4096 -C "你的邮箱地址"
    • -t rsa：指定密钥类型为 RSA。
    • -b 4096：设置密钥长度为 4096 位（更安全）。
    • -C：添加注释（通常为邮箱，用于标识密钥）
  • cat ~/.ssh/id_rsa.pub 打印创建的公钥
  • cat ~/.ssh/id_rsa.pub | pbcopy 拷贝公钥
    把公钥粘贴到仓库后验证
  • ssh -T code.byted.org 验证成功会出现：
  • *** Welcome to Gitlab, 你的名字 ***

shell 相关

cat /etc/shells 查看当前终端装了哪些shell
zsh --version 可查看某个shell的版本
echo $SHELL 查看当前终端使用的是哪个shell
chsh -s /bin/bash 改变某个shell
set -e 脚本开头部分写此命令，表示脚本中其他命令返回值非0则立即退出shell，0表示命令执行正常。
set +e 表示脚本中其他命令返回值非0时也继续向下执行，bash默认就是set +e
go2shell：/Applications/Go2Shell.app/Contents/MacOS 将Go2ShellHelper.app 按住Command键同时拖动拖到 Finder 工具栏上。

grep

  • grep -v 'abc' 查找不含有abc的行
  • grep -v '^$' old.txt 过滤old.txt文本中的空白行
  • grep -v '^abc' xyz.txt 不显示xyz.txt文本中以abc开头的行
  • grep -iE "abc|def" 过滤abc或者def
  • grep -rn '_av_ac3_parse_header' /path/to/ffmpeg/source 找到所有包含这个符号的文件

sed

  • cat demo.txt | sed "s/'//g" 表示在把文本中的单引号去掉
    • s：表示替换操作
    • /‘/：两个斜杠之间表示要匹配的字符，此处表示匹配单引号
    • /‘/*/：表示把单引号替换为星号
    • g：表示全局操作

awk

  • cat demo.txt | awk '{print $3}' 表示打印出文本中的第三列

Permission denied

sudo chmod -R 750 /Users/bytedance/Downloads
其中
-R 是指级联应用到目录里的所有子目录和文件
750 表示文件所有者有读、写、执行权限，文件组有读、执行权限，其他用户没有权限
属主：rwx=4+2+1=7。
属组：r-x=4+0+1=5。
其他：—=0+0+0=0

SpotLight：

1、输入关键词，接着同时点击 cmd + B，系统会马上打开默认浏览器用默认的搜索引擎进行搜索
2、输入关键词，找到你需要的那个文件，再按住 cmd 键，系统会展示该文件的详细路径；点击 cmd +R系统会打开该文件所在的文件夹。
3、输入英文单词后，同时点击 cmd + L，搜索框会自动定位至单词翻译功能;点击 cmd + D，还会自动打开字典应用，显示该单词的解释.
4、输入 1美元、9欧元、102英里、97英尺、39.1英寸、209盎司等等，系统会自动转换为人民币、人民币、公里、厘米、厘米、克等单位；
5、输入 1+1，会自动得出结果 2，可进行计算。

Git命令相关

  • 子模块更新：git submodule update --init --recursive
  • 带用户名克隆：git clone http://username:password@gitee.com/damn_code/SubProj1.git
  • 当前所有分支： git branch -a
  • 过滤某个分支： git branch -a | grep xxxx
  • 创建新的分支：git branch dev_branch
  • 切换到dev_branch：git checkout dev_branch
  • 这两条命令可以合并为一条：git checkout -b dev_branch
  • 合并：先切换到要合并的那条分支上，再合并。比如我要把dev_branch合并到master上：git checkout master //切换到master主干
  • 合并dev_branch: git merge dev_branch
  • 删掉完成任务的本地分支: git branch -d dev_branch (要是大写-D，则代表强制删除，则该分支修改的数据可能会丢失)
  • 删除远程分支: git push origin --delete dev_branch
  • 第一次把本地的新分支推送到远程: git push --set-upstream origin dev_branch
  • 查看各个分支最后一次的提交 git branch -v
  • 查看哪些分支已被合并到当前分支（也就是说那些分支是当前分支的直接上游）git branch --merged
  • 查看哪些分支尚未与当前分支合并:git branch --no-merged
  • 强制提交到远程，覆盖远程分支: git push origin master --force
  • 查看提交历史：git log (会按提交时间列出所有的更新)
  • 查看提交的内容差异：git log -p -2 （-2表示仅显示最近的两次更新）
  • 仅查看提交记录，看不到diff：git log -2 （-2表示仅显示最近的两次更新）
  • 将每个提交放在一行显示：（还有short/full/fuller展示的信息或多或少）
    git log --pretty=oneline -2
  • 打tag:（在本地commit完成之后打tag）git tag -a 1.0.1 -m "1.0.1版本稳定版发布"
  • 把tag推送到远程:git push —-tags
  • 删除本地tag：git tag -d 0.1.3
  • 删除远程tag：git push origin --delete tag 1.0.2
  • 拉取新的远程分支，并切换到该分支:git checkout -t remotes/origin/develop
  • 本地分支和远程分支关联起来:git branch --set-upstream-to=origin/remote_branch your_branch
  • 保留改动撤销commit: git reset --soft commit_id
    • 假如某个commit丢失了，想找回这个commit改动的代码：
    • git reflog: 是显示最近的git操作
    • 找到符合最近的改动的操作对应的commit的id
    • 执行git reset –soft commit_id，这个命令能就会恢复刚才被误删的代码，并且当前分支有其他改动，也能恢复之前的丢失的commit
  • 撤销–depth：git fetch –unshallow
  • 回退版本:
    • git reset --hard commit_id 回退到之前某个版本，但中间的改动不保留
    • git reset --soft commit_id 回退到之前某个版本，但中间的改动会保留
    • git revert -n commit_id 反做之前的某个commit
  • 先在git仓库添加文件，然后才添加.gitignore忽略文件，导致.gitignore不生效。因为 .gitignore 只能忽略那些没有被追踪(track)的文件，因为 git 存在本地缓存，如果文件已经纳入了版本管理，那么修改 .gitignore 是不能失效的
    • git rm -r –cached .
  • git remote prune origin 是一个Git 命令，用于从本地Git 仓库中删除与远程仓库（origin）不再存在的远程跟踪分支。 在Git 中，远程跟踪分支是本地Git 仓库中用于跟踪远程仓库的分支。
  • 设置全局git仓库的账户 git config --global user.email "your Email.@example.com"
  • 设置全局git仓库的用户名 git config --global name "your name"
  • 打印全局git仓库账号 git config --global user.email
  • 打印全局git仓库用户名 git config --global user.name
  • 设置当前git仓库的账户 git config user.email "your Email.@example.com"
  • 设置当前git仓库的用户名 git config user.name "your name"
  • 打印当前git仓库账号 git config user.email
  • 打印当前git仓库用户名 git config user.name
  • 清除设置的全局用户名和邮箱 git config --global --unset user.email
  • push 报错内容：send-pack: unexpected disconnect while reading sideband packet
    • 出现这种情况有可能是缓存过小引起的，增加缓存即可：git config --global http.postBuffer 1048576000
    • 单位是Byte, 上面的 1048576000B 就是 1G

Xcode 相关

  • LLDB命令 由内存地址获取该地址对应的对象名称
    • p ((id)0x00000001017921d0)->isa
  • 打印Xcode
    • xcode-select –print-path
  • 更改默认Xcode
    • sudo xcode-select -switch /Applications/Xcode.app/
  • Command CompileSwiftSources failed with a nonzero exit code
    • 通过在Build Setting里面自添加一条user-defined(备注：点击Level 边上的 + 号就可以添加)，属性为SWIFT_ENABLE_BATCH_MODE,值为NO。再次编译就能发现编译失败的真正原因。
  • 符号化
    • 找到对应的crash日志和对应的dsym文件夹
    • 打印下Xcode地址：xcode-select -print-path
    • 重新reset下DEVELOPER_DIR地址：export DEVELOPER_DIR=/Applications/XCode.app/Contents/Developer
    • 在如下地址找到 symbolicatecrash 应用程序，拷贝到一个新的空目录中，并且吧crash日志和dsym文件放入到新目录中
      • /Applications/Xcode.app/Contents/SharedFrameworks/DVTFoundation.framework/Versions/A/Resources/symbolicatecrash
    • 输入如下命令即可解析：./symbolicatecrash AwemeInhouse.crash AwemeInhouse.app.dSYM > CrashLog.crash
  • 终端展示所有 build settings
    • xcodebuild -showBuildSettings

Sublime

  • 正则匹配规则
    • . 匹配除换行符以外的任意字符
    • \w 匹配字母或数字或下划线
    • \s 匹配任意的空白符
    • \d 匹配数字
    • \b 匹配单词的开始或结束
    • ^ 匹配字符串的开始
    • $ 匹配字符串的结束
    • *重复零次或更多次
    • +重复一次或更多次
    • ? 重复零次或一次
    • n 重复n次
    • {n,} 重复n次或更多次
    • {n,m} 重复n到m次
  • \r\n 替换
    • 在搜索栏中输入\\r\\n，变化匹配到字符串中的\r\n
    • 在替换栏中输入\n，即可换行替换。
  • 模糊查找、替换
    • 比如查找以AAAA开头、以BBBB结尾的字符串
      • (AAAA.*BBBB)
  • 查找文件
    • command + p，然后输入文件名
  • 查找文件中的函数名
    • command + p，然后输入@符号，再接着输入名称
  • 定位到某一行
    • command + p，然后输入:冒号，再接着输入行号
