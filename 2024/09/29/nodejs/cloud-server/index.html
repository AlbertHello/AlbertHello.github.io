云服务器与 Linux 命令

本文最后更新于 2026-05-20T21:10:39+08:00

Linux 命令

登录

普通登录

运行 ssh name@server 然后输入密码即可登录，name 默认为 root

登录之后，执行 exit 即可退出

建立信任

第一，在你个人电脑生成 ssh key（不了解的可以搜索“widows 搜索 ssh key”，步骤很简单）

生成之后有 id_rsa 和 id_rsa.pub 两个文件，其中 id_rsa 是机密文件，不要外泄！

第二，拷贝下 id_rsa.pub 的文件内容，粘贴到服务器 ~/.ssh/authorized_keys 文件中（如没有该文件，就创建一个）

PS：可以粘贴多个客户端的 id_rsa.pub 内容。

第三，退出，重新登录，就不用再输入密码了。

注意，如果配置之后还需要输入密码，可以先密码登录，然后尝试如下操作

cd .ssh
chmod 700 ../
chmod 700 .
chmod 600 authorized_keys

删除目录 rm -rf

拷贝目录 cp

创建文件 touch vi
删除文件 rm
修改文件名 mv
拷贝文件 cp

文件内容操作

查看文件 cat head tail
编辑文件内容 vi

远程操作

远程拷贝 scp
把当前目录 package.json 拷贝到服务器 /root/test-dir 目录下。

如果设置了信任，则不用输入密码。否则得需要输入密码。

scp ./package.json root@125.124.59.115:/root/test-dir
远程执行命令
远程执行命令 cd ./test-dir; touch a.txt

ssh root@125.124.59.115 “cd ./test-dir; touch a.txt”

CentOS 安装软件

以 root 用户登录服务器控制台。

确认是否可以访问外网

使用 ping 命令测试，是否可以正常访问外网。
ping www.baidu.com
回车之后，得到这样的结果是正常的：

64 bytes from 39.156.66.14: icmp_seq=1 ttl=52 time=27.697 ms
64 bytes from 39.156.66.14: icmp_seq=2 ttl=52 time=21.515 ms
64 bytes from 39.156.66.14: icmp_seq=3 ttl=52 time=21.330 ms
64 bytes from 39.156.66.14: icmp_seq=4 ttl=52 time=20.995 ms
64 bytes from 39.156.66.14: icmp_seq=5 ttl=52 time=21.072 ms

ctrl + c 结束测试

如不能正常访问外网，需要查看云服务器配置。不明白的可以问问他们的客服人员，一般在网站右下角都有客服入口。

安装 Git

使用 yum 安装

sudo yum install git

待安装完成之后，查看 git 版本

git –version

安装 nginx
使用 yum 安装

sudo yum install -y nginx

待安装完成之后，查看 nginx 版本

nginx -Vs

安装 nodejs
分别执行如下命令，安装 nodejs 16

curl -fsSL https://rpm.nodesource.com/setup_16.x | sudo bash -

sudo yum install -y nodejs

待安装完成之后，查看 nodejs 版本

node -v

npm -v

然后全局安装 pm2

npm i -g pm2

安装 wget

wget 可以在控台访问一个 url 地址，并得到返回结果，用于测试 web server 是否正常运行。

yum -y install wget

待安装完成之后，查看 wget 版本

wget -V
