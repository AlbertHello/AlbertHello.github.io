Mac 安装 FFmpeg

本文最后更新于 2026-05-20T21:10:39+08:00

mac 编译ffmpeg

环境：2022款Mac Book Air 13寸 M2芯片，ffmpeg-6.0.1 和 ffmpeg-4.3.2

2022款Mac Book Air 13寸 M2芯片的mac

自动安装、编译

  • brew install ffmpeg
    • 这是最直接的安装ffmpeg库的方法
    • 安装的ffmpeg库的目录在/opt/homebrew/Cellar/ffmpeg/
    • ffmpeg/ffplay/ffprobe原身都在/opt/homebrew/Cellar/ffmpeg/对应的ffmpeg版本/bin/目录下
    • ffmpeg/ffplay/ffprobe替身都在/opt/homebrew/bin/目录下，终端直接敲命令也是用的该目录下的可执行程序

手动编译

  • brew install yasm
    • ffmpeg的编译过程依赖yasm
    • 若未安装yasm会出现错误：nasm/yasm not found or too old. Use –disable-x86asm for a crippled build.
  • brew install sdl2
    • ffplay依赖于sdl2
    • 如果缺少sdl2，就无法编译出ffplay
  • brew install fdk-aac
    • 不然会出现错误：ERROR: libfdk_aac not found
  • brew install x264
    • 不然会出现错误：ERROR: libx264 not found
  • brew install x265
    • 不然会出现错误：ERROR: libx265 not found
    • 其实x264、x265、sdl2都在曾经执行brew install ffmpeg的时候安装过了。
  • 可以通过brew list的结果查看是否安装过
    • brew list | grep fdk
    • brew list | grep x26
    • brew list | grep -E ‘fdk|x26’
  • 如果已经安装过，可以不用再执行brew install

configure

  • 解压根目录中的ffmpeg-6.0.1源码和ffmpeg_rtmp_support_h265-6.0的压缩包
  • 把 ffmpeg 支持 265 改动的三个文件拷贝到 ffmpeg源码中的 libavformat 目录，替换源文件，
  • cd 到 ffmpeg-6.0.1 目录，也就是 ffmpeg 源码目录，然后按照下面命令开始编译。
  • 编译命令
    • ./configure –prefix=/usr/local/ffmpeg –enable-shared –disable-static –enable-gpl –enable-nonfree –enable-libfdk-aac –enable-libx264 –enable-libx265
  • –prefix
    • 用以指定编译好的FFmpeg安装到哪个目录
    • 一般放到/usr/local/ffmpeg中即可
  • –enable-shared
    • 生成动态库
  • –disable-static
    • 不生成静态库
  • –enable-libfdk-aac
    • 将fdk-aac内置到FFmpeg中
  • –enable-libx264
    • 将x264内置到FFmpeg中
  • –enable-libx265
    • 将x265内置到FFmpeg中
  • –enable-gpl
    • x264、x265要求开启GPL License
  • –enable-nonfree
    • fdk-aac与GPL不兼容，需要通过开启nonfree进行配置

在配置ffmpeg-6.0.1时一切正常，在配置ffmpeg-4.3.2时，执行上述的编译命令抱错：大概率是mac系统版本、brew install的那些依赖库版本太新，和旧版本的ffmpeg不兼容，而且最终make install也没有产生ffplay

  • ERROR: libfdk_aac not found
    • 解决办法：在–enable-libfdk_aac后面带上libfdk_aac头文件和库文件路径
    • –enable-libfdk-aac –extra-cflags=”-I/opt/homebrew/Cellar/fdk-aac/2.0.3/include” –extra-ldflags=”-L/opt/homebrew/Cellar/fdk-aac/2.0.3/lib”
  • ERROR: libx264 not found
    • 解决办法：在–enable-libx264后面带上x264的头文件和库文件路径
    • –enable-libx264 –extra-cflags=”-I/opt/homebrew/Cellar/x264/r3108/include” –extra-ldflags=”-L/opt/homebrew/Cellar/x264/r3108/lib”
  • ERROR: x265 not found using pkg-config
    • 解决办法：brew install pkg-config解决

编译

接下来开始解析源代码目录中的Makefile文件，进行编译。-j8表示允许同时执行8个编译任务，启用8个cpu 核。也可以看下自己的电脑有几个核：

  • 查看物理 CPU 核数：sysctl hw.physicalcpu
  • 查看逻辑 CPU 核数：sysctl hw.logicalcpu
  • 查看硬件信息总览：system_profiler SPHardwareDataType
  • 编译：make -j8 利用8核，加快编译速度

安装

  • 安装：sudo make install 此命令会把ffmpeg安装到指定的位置：/usr/local/ffmpeg

安装完毕后，/usr/local/ffmpeg的目录结构如下所示:

配置PATH

为了让bin目录中的ffmpeg、ffprobe、ffplay在任意位置都能够使用，需要先将bin目录配置到环境变量PATH中。编辑.zprofile 或者 .zshrc、或者 .bashrc

  • 写到配置文件中：export PATH=/usr/local/ffmpeg/bin:$PATH
  • 重载环境配置文件：source ~/.zprofile
