比如：
res.setHeader('Set-Cookie', userid=xxx; path=/; httpOnly; expires=${getCookieExpires()})

  • 服务端修改cookie，通过response 中的 header 传递回来
  • key：Set-Cookie
  • value：就是修改后的 cookie 字符串
    • path=/ 表示域名根目录，只要是该域名登录，所有网页通用登录信息
    • httpOnly 表示只允许后端修改cookie
    • expires 设置 cookie 过期时间

nginx 反向代理

  • 反向代理就是说对于客户端来说是不可见的，代理行为发生在服务端。
    • 当客户端请求 localhost/index.html 页面时，请求是打到nginx 服务，然后根据配置，再把请求定向到不同的服务。
  • 正向代理就好比客户端主动使用 vpn 软件访问。
  • M2 Mac 的nginx 配置目录在/opt/homebrew/etc/nginx/nginx.conf

前后端联调

需要在终端运行多个命令：

  • 1、首先终端运行redis-server，启动redis
  • 2、我们还需要一个前端页面才能展示博客的数据，所以在 6.http-test/目录下输入http-server -p 8001 命令，运行起来 httpserver
  • 3、再执行 nginx 命令，让 nginx 服务跑起来，nginx 监听的是 8080 端口。
  • 4、再在 /nodejs/nodejs/2.blog/blog1/ 目录下运行npm run dev 命令，跑起来nodejs 后端服务。后端服务监听的是 2100 端口。
  • 5、最后可以在浏览器输入localhost:8080/index.html，这是往nginx 服务发起请求。nginx 根据配置的nginx.confi就会往localhost:8001或者localhost:2100再次发起请求，获取博客相关数据。
