RTP-AAC格式

本文最后更新于 2026-05-20T21:10:39+08:00

RTP 传输 AAC 音频

AAC在RTP传输时有2种格式，分别是mp4a-latm和 mpeg4-generic

mp4a-latm

mp4a-latm 参考 rfc6416标准

SDP音频部分：

  • cpresent=0表示rtp包里面不包含config头。表示带外传输音频格式，节省带宽
  • config是aac的基本信息，如samplerate，channels，frame length等信息。
  • profile-level-id:
    • 1 : Main Audio Profile Level 1
    • 9 : Speech Audio Profile Level 1
    • 15: High Quality Audio Profile Level 2
    • 30: Natural Audio Profile Level 1
    • 44: High Efficiency AAC Profile Level 2
    • 48: High Efficiency AAC v2 Profile Level 2
    • 55: Baseline MPEG Surround Profile (see ISO/IEC 23003-1) Level 3
  • config是StreamMuxConfig的16进制字符串。具体streamMuxConfig参考14496-3标准

config 解释

上面例子40 00 57 24 10 00 3f c0表示，具体含义如下：

  • 0100 0000: 第一位固定0，第二位1表示所有frame都是同样的时间信息，后面6位表示subFrame是0，是表示有1个音频流。
  • 0000 0000: 前4位0000表示numProgram=0, 接下来3位000，表示numLayer=0，之后的bit表示AudioSpecificConfig
  • 0101 0111 audioObjectType: 5 bits.
    • 主流的 AAC Main=1
    • AAC_LC=2
    • AAC_SSR=3
    • SBR=5
    • PS=29
  • 0010 0100 samplingFrequencyIndex: 4 bits，例子里是7，代表22050Hz。这是一个数组的索引值index，有个采样率的数组，根据index查询采样率。
  • channelConfiguration: 4 bits，例子里是2，表示双声道。

之后数据表示GASpecificConfig，GASpecificConfig结构为：
frameLengthFlag: 1 bit，0: 1024， 1:960
depensOnCoreCoder: 1bit
extensionFlag: 1bit
extensionFlag3: 1bit

此时还剩下0x3f, 0xc0,
frameLengthType: 3 bits，例子里是1，如果是1，则frameLength=9 bits
frameLength: 9 bits, 全1，表示xx
后面全0.

RTP latm 封装 aac 格式：

+———-+—————-+
|RTP Header| AudioMuxElement|
+———-+—————-+
<–RTP Payload—>

AudioMuxElement: ｜PayloadLengthInfo｜PayloadMux｜

RTPHeader里面的
PayloadLengthInfo：前面几个字节是后面es的长度，第一个不是0xff的字节结束，例如0xff 0x10, 长度为0xff+0x10, 0xff 0xff 0x10,那么长度是0xff + 0xff + 0x10
PayloadMux： 就是裸aac es数据

It is RECOMMENDED to put one audioMuxElement in each RTP packet. If the size of an audioMuxElement can be kept small enough that the size of the RTP packet containing it does not exceed the size of the Path MTU, this will be no problem. If it cannot, the audioMuxElement SHALL be fragmented and spread across multiple packets.
目前应该都是一个RTP足够装下一个audio数据，如果超过，分为多个RTP包，通过Marker字段来知道结尾。
Marker (M) bit: The marker bit indicates audioMuxElement boundaries. It is set to 1 to indicate that the RTP packet contains a complete audioMuxElement or the last fragment of an audioMuxElement.

举例

  • config = 400057241056e2a000
    4 0 0 0 5 7 2 4 1 0 5 6 e
  • 0100 0000 0000 0000 0101 0111 0010 0100 0001 0000 0101 0110 1110
    下面对上面的二进制进行解释：
  • 0 1bit 固定是 0
  • 1 1bit 表示所有的帧都是同样的时间信息
  • 000000 6bits subframe = 0，表示有一个音频流
  • 0000 4bits numProgram = 0，
  • 000 3bits
  • 00101 5bits: aot = 5
  • 0111 4bits: samplerateIndex = 7, 即22050
  • 0010 4bits: channel = 2
  • 0100 4bits: samplerateIndex = 4, 即44100
  • 0 1bit, 0 帧长1024，1 帧长 960
  • 0 1bit，
  • 0 1bit
  • 1 1bit
