webrtc 的线程

rtc::Thread

  • webrtc中的Thread是一个对象，不是一个线程，对象里会有真正线程的句柄
  • wertc中的线程是有队列的
  • webrtc中的线程是以事件驱动的。即收到某哥事件时才会去队列中取消息，根据消息做具体的动作。

ThreadManager

主要成员：

解释一下key_的作用：

  • ThreadManager有两个队列，一个是存储thread对象指针的队列，一个是线程私有存储队列（可以这么理解）。
  • 比如一个有全局变量，多线程环境下，每个线程都可以取访问、修改，但如果每个线程只想单独拥有它，自己线程修改的该全局变量的值，只允许自己线程中的函数访问，不希望其他线程来访问自己的值，那么这就需要创建线程的私有数据，这个数据会屏蔽其他线程，但全句变量还是那一个全局变量。

message_queues_ 和 key_

  • message_queue中的thread对象和thread local storage map 表中的 thread 对象对应
  • message_queues_ 队列存储thread对象指针。
    • thread对象构造时就会被加入到该队列中
  • thread local storge队列这是系统的，是个map表，key-value。和 message_queues_ 中的元素一一对应。
    • 线程执行时，当调用SetCurrentThread，当前线程与thread对象绑定，又调用ThreadManager的SetCurrentThreadInternal，最终调用pthread_setspecific存储起来。

thread 构造函数：

Thread::DoInit

ThreadManager::AddInternal

  • 当Thread对象构造时，最终调用了ThreadManager::AddInternal
  • 在AddInternal 函数中把 Thread 对象指针加入到了ThreadManager 的 message_queues_ 队列中

ThreadManager::SetCurrentThread

  • 该函数会把当前线程和传入的Thread对象进行绑定。
