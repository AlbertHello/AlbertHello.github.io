使用代码连接 mysql 数据库时报错：

ER_NOT_SUPPORTED_AUTH_MODE

解决一：在 workbench 中执行语句：
ALTER USER 'root'@'localhost' IDENTIFIED WITH mysql_native_password BY 'Wang-920416';
当然这是本地数据库有密码的设置。这不是彻底的解决办法。

解决二：
从 MySQL 8.0.4 开始，MySQL 默认身份验证插件从 mysql_native_password 改为 caching_sha2_password 所导致。
执行：

npm list mysql

这个不是服务器MySql数据库的版本，指的是const mysql = require(‘mysql’);中的mysql连接库。npm install mysql2 –save 安装mysql2，代码中使用mysql2。

再次运行node index.js就正常了
