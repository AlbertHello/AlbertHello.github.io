webrtc之BasicPortAllocator+BasicPortAllocatorSession

本文最后更新于 2026-06-23T17:28:25+08:00

对 BasicPortAllocator 的认识：

  • 用于端口分配的类：PortAllocator，实际使用的是它的子类：BasicPortAllocator。

  • BasicPortAllocator 对象是在 PeerConnectionFactory::CreatePeerConnectionOrError 函数中创建的，也就是创建 peerconnection 的前一步创建的。参考 webrtc 之创建 peer_connection

  • 同样在 peerconnection 初始化的时候 BasicPortAllocator 也初始化。

  • BasicPortAllocator 的构造函数核心入参是 Environment、BasicNetworkManager、BasicPacketSocketFactory、TurnCustomizer，其中网络管理和 socket 工厂最关键。

  • 创建 JsepTransportController 时，BasicPortAllocator 被当做入参传入，JsepTransportController 在创建P2PTransportChannel 时，再把 BasicPortAllocator 传给 p2p。

  • p2p 在 MaybeStartGathering() 函数中通过 BasicPortAllocator 调用 PortAllocator::CreateSession，函数内部再调用 BasicPortAllocator::CreateSessionInternal 函数，在该函数内创建 BasicPortAllocatorSession，BasicPortAllocatorSession 也持有了 BasicPortAllocator。BasicPortAllocatorSession也是一个子类，其继承自PortAllocatorSession。

对 BasicPortAllocatorSession 的认识：

  • 它也是一个子类，继承自 PortAllocatorSession。

  • 它是在 BasicPortAllocator::CreateSessionInternal() 函数创建的，是 p2pchannel 的 MaybeStartGathering 函数通过持有的 BasicPortAllocator 触发创建的。

  • BasicPortAllocatorSession 的构造函数中通过持有的 BasicPortAllocator 拿到了 BasicNetworkManager，调用 BasicNetworkManager::StartUpdating()。M146 中通过 Thread::PostTask(UpdateNetworksContinually) 触发首次网卡枚举，然后每 2 秒 PostDelayedTask 一次。

  • p2p 在创建 BasicPortAllocatorSession 后，再调用 BasicPortAllocatorSession::StartGettingPorts() 函数。M146 中该函数直接 PostTask(GetPortConfigurations)，配置 ready 后再 PostTask(OnAllocate)，开始触发 port allocate 机制。

  • 它的 OnAllocate 中调用 DoAllocate，遍历 Networks，为每个需要分配端口的 Network 创建对应的 AllocationSequence 对象，AllocationSequence 详见 webrtc 之 AllocationSequence 详解

创建 BasicPortAllocator

在创建 peerconnection 时创建 BasicPortAllocator

  • PeerConnectionFactory::CreatePeerConnectionOrError

    • BasicPortAllocator::BasicPortAllocator //

      • BasicPortAllocator::InitRelayPortFactory // 最主要的就是创建default_relay_port_factory_

初始化 BasicPortAllocator

在初始化 PeerConnection 时初始化 PortAllocator

  • PeerConnection::InitializeNetworkThread // PeerConnection 初始化网络线程

    • PeerConnection::InitializePortAllocator_n // 切换到网络线程

      • PortAllocator::Initialize // 父类初始化

      • PortAllocator::set_flags // 父类初始化后配置一些 flag，进行定制

        • 比如设置是否开启 tcp candidate 收集、是否收集 ipv6 类型的、是否只收集 relay 类型的等等

      • PortAllocator::SetConfiguration // 配置stun/turn server 信息

Port Allocate 任务机制

M146 中 BasicPortAllocatorSession 已经改成 network_thread_->PostTask() 驱动。

    S->>T: PostTask(GetPortConfigurations)

    S->>Seq: new AllocationSequence
    S->>Seq: Init()

    T->>Seq: Process(UDP/Relay/TCP)

    Port-->>S: OnCandidateReady / OnPortComplete
    S-->>P2P: NotifyCandidatesReady / NotifyPortReady

  • StartGettingPorts

    • 设置 SessionState::GATHERING，然后 network_thread_->PostTask(GetPortConfigurations)。

  • GetPortConfigurations / ConfigReady / OnConfigReady

    • 创建 PortConfiguration，其中包含 STUN/TURN server、username、password 等信息。
    • ConfigReady 再投递 task 到 OnConfigReady，OnConfigReady 保存 config 并调用 AllocatePorts()。

  • AllocatePorts / OnAllocate

    • AllocatePorts 投递 OnAllocate(allocation_epoch)。
    • OnAllocate 检查 epoch，确认 network manager 已启动后调用 DoAllocate()。

  • DoAllocate

    • 读取 NetworkManager::GetNetworks()。
    • 过滤 IPv6、link-local、costly network、ignore mask。
    • 为每个 Network 创建 AllocationSequence，依次调用 Init() 和 Start()。

  • AllocationSequence::Start / Process

    • Start() 投递 Process()。
    • Process() 按 UDP -> Relay -> TCP 阶段创建端口，阶段之间用 PostDelayedTask。

  • OnAllocationSequenceObjectsCreated

    • DoAllocate 最后会 PostTask(OnAllocationSequenceObjectsCreated)，用于检查是否可以发出 candidate 收集完成信号。
